argument-hint: [PR 제목 힌트 또는 --draft] (선택)
allowed-tools: Bash, Read
---

`$ARGUMENTS` 에 `--draft` 가 있으면 초안 PR로, 제목 힌트가 있으면 제목 생성에 반영한다.

### 0. 사전 점검
- 현재 브랜치가 기본 브랜치(`main`/`master`)면 중단하고 `/branch` 를 먼저 쓰라고 안내한다.
- 커밋되지 않은 변경이 있으면 알리고, 포함할지 사용자에게 확인한다.

### 1. 푸시
현재 브랜치를 업스트림으로 푸시한다 (`git push -u origin <branch>`).

### 2. 제목·본문 생성
`git log <base>..HEAD` 의 커밋들을 근거로:
- **제목**: 변경 핵심을 한 줄로 (단일 커밋이면 그 요약 재사용 가능). 초안이면 `WIP: ` 접두.
- **본문**:
  ```
  ## Summary
  - 무엇을 왜 바꿨는지 1~3줄

  ## Test plan
  - 어떻게 검증했는지 / 검증 방법

  ## Notes
  - 리뷰어가 알아야 할 주의점 (없으면 생략)
  ```

### 3. PR 생성
- `gh` CLI 가 있으면 `gh pr create` 로 생성한다 (`--draft` 옵션 반영).
- 없으면 푸시까지만 하고, 호스트의 "compare" URL 또는 생성된 제목·본문을 출력해
  사용자가 수동으로 열 수 있게 한다.
- **머지는 절대 자동으로 하지 않는다.** 생성된 PR URL을 보고한다.
